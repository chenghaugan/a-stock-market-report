**用途**：由 Hermes Agent 在完成数据采集后加载，用于驱动 Tavily 研究和完整 Markdown 报告生成。

**前置条件**：已完成 `run_report.py` 数据采集，JSON 数据文件位于 `{workdir}data_YYYYMMDD.json`。

## 报告信息

**⚠️ 以下变量在执行时必须替换为实际值**：
- `{date}` → 报告日期，格式 `YYYY-MM-DD`（如 `2026-04-19`）
- `{target_date}` → 数据日期，同上
- `{report_type}` → 报告类型：`日报`、`周报` 或 `月报`
- `{workdir}` → 工作目录：`{data_dir}`
- `{generated_at}` → 生成时间戳

```
今日日期（北京时间）：{date} 15:00 收盘后
报告类型：{report_type}  # daily→日报 / weekly→周报 / monthly→月报
工作目录：{workdir}  # {data_dir}
JSON数据文件：{workdir}data_{date}.json（日期用数字格式：data_20260419.json）
Obsidian推送目标：
  - URL: https://obsidian-api.870314.top:5000
  - Vault: /01_投资研究/每日复盘
  - 子目录: {subdir}  # 日报→日报 / 周报→周报 / 月报→月报（自动推断）
  - 完整路径: /01_投资研究/每日复盘/{subdir}/{date}_{report_type}.md
  - 文件名: {date}_{report_type}.md（例如：2026-04-19_日报.md）
  - 推送方式: 推荐使用 push_to_obsidian.py（自动处理子目录），而非原始 curl

**📊 数据显示格式说明（重要）**：
- `hot_sectors[].change` 和 `sectors[].change`：**已统一为百分比格式**（fetch_sectors 中已做归一化：新浪小数 0.4996 → 0.50）。直接使用 change 字段的值，按下方示例格式显示。
- 指数/个股涨跌幅：已经是百分比值（如 1.42 表示 1.42%），显示格式 `{value}%`
- 板块涨跌幅显示规则：
  - `change: 0.50` → 显示 `+0.50%`（正数）或 `-{abs(value)}%`（负数）
  - `change: 85.93` → 显示 `+85.93%`（已是百分比，无需转换）
- 模板中的 `{pct}` 占位符：填写 change 字段的绝对值（不加 ±），表格单元格自动带正负号
- **绝对不要**对板块 change 值再做 ×100 运算，JSON 中已经是百分比格式
```

```markdown

## 二、重要财经要闻

**宏观环境**：{宏观背景描述，如：三大指数走势分化，上证终结五连涨，创业板创近11年新高，反映投资者对某事件的审慎态度}

**重点消息**：

*   {判断1}

*   {判断4}

## 三、五大热门板块深度解析

**板块筛选逻辑**：从投资经理视角综合考量，非单纯按涨跌幅排序。权重分配：
- 板块热度（30%）：市场讨论度、新闻曝光度、社交媒体热度
- 资金关注度（25%）：板块资金净流入、成交量放大程度
- 涨跌幅（25%）：当日涨幅绝对值
- 持续性（20%）：连续上涨天数、前期表现、趋势强度

| 排名 | 板块 | 涨跌幅 | 代表个股 | 逻辑推演 |
| --- | --- | --- | --- | --- |
| 1 | {板块名} | {pct}% | {2-3个代表个股，如"XX股份、YY集团"} | {100-150字深度分析：为何入选（热度/资金/持续性）、行业逻辑、政策催化、资金动向、产业链传导、后续展望} |
| 2 | {板块名} | {pct}% | {2-3个代表个股} | {100-150字深度分析} |
| 3 | {板块名} | {pct}% | {2-3个代表个股} | {100-150字深度分析} |
| 4 | {板块名} | {pct}% | {2-3个代表个股} | {100-150字深度分析} |
| 5 | {板块名} | {pct}% | {2-3个代表个股} | {100-150字深度分析} |

**⚠️ 注意**：
- 代表个股需是该板块的龙头股、涨停股或资金关注度高的个股
- 代表个股数量2-3个，简洁列出名称即可
- 涨跌幅仅为参考指标之一，不是唯一排序依据

## 四、五大核心个股追踪

**选股逻辑**：从全市场热点个股中筛选，热门板块权重60% + 个股权重40%，剔除新股（上市不足30天）。优先选择今日热门板块的龙头股、涨停股、资金关注度高的个股。

| 排名 | 个股 | 涨跌幅 | 所属板块 | 板块地位 | 逻辑推演 |
| --- | --- | --- | --- | --- | --- |
| 1 | {个股名} | {pct}% | {所属热门板块} | 板块龙头 | {100-150字深度分析：为何入选、基本面、技术面、资金面、板块联动、后续展望} |
| 2 | {个股名} | {pct}% | {所属热门板块} | 涨停核心标的 | {100-150字深度分析} |
| 3 | {个股名} | {pct}% | {所属热门板块} | 资金聚焦标的 | {100-150字深度分析} |
| 4 | {个股名} | {pct}% | {所属热门板块} | 龙头股 | {100-150字深度分析} |
| 5 | {个股名} | {pct}% | {所属热门板块} | 涨停核心标的 | {100-150字深度分析} |

**⚠️ 注意**：
- 个股必须来自第三章热门板块（板块权重60%）
- 非热门板块的个股仅作为补充（个股权重40%）
- 剔除上市不足30天的新股
- 来源是全市场（非自选股范围）
- 涨停池数据可作为参考，但不等于直接取涨停池TOP5
```

## Tavily 研究策略

### 搜索原则

1. **搜索次数**：每次报告至少执行 **3-5 次** Tavily 搜索，保证信息完整性
2. **搜索深度**：`search_depth=basic`（常规），复杂行情用 `search_depth=advanced`
3. **时间范围**：`time_range=day`（日报）、`time_range=week`（周报）、`time_range=month`（月报）
4. **主题**：`topic=finance`

### 搜索 Query 组合

#### 日报必搜（至少3次）

```
Query 1: "A股 今日行情 {date} 大盘 走势"

```

## 报告写作规范

### 语言风格
- 分析师风格：专业、简洁、数据驱动
- 避免口语化表达（如"感觉"、"好像"、"据说"）
- 禁止使用："由AI深度分析后补充"、"待更新"、"暂无数据时请补充"等占位符
- **如果某项数据确实为空**，如实标注"该数据暂无来源"，不要使用占位符

### 数据使用
- 优先使用 JSON 文件中的准确数字
- 涨跌幅保留2位小数（%）
- 成交额用"亿元"单位
- 点位取整数

### 禁止事项
- 禁止生成任何占位符（报告必须完整可读）
- 禁止编造未经验证的数据
- 禁止提供具体投资建议（只能说"关注xxx方向"）
- **禁止省略表格分隔行**：所有 markdown 表格（无论章节级别还是子章节级别）必须包含完整的 `| 表头 |` + `| --- |` + `| 数据 |` 三行结构，AI 不得跳过分隔行直接生成数据行

## ⚠️ 输出前强制自检清单

生成报告后，**必须逐项检查**，任何一项不通过都不得推送：

```
□ 第一章"市场全景概览"存在，包含四大指数表格
□ 第二章"重要财经要闻"存在，包含宏观环境 + 至少3条新闻 + 市场影响判断
□ 第三章"五大热门板块深度解析"存在，包含TOP5板块表格，每板块分析≥100字
□ 第四章"五大核心个股追踪"存在，包含涨停池TOP5个股表格，每个股分析≥100字
□ 第五章"自选股跟踪"存在，包含小结 + 异动表格
□ 第六章"策略总结与推演"存在，字数≥300字
□ 无"由AI深度分析后补充"等占位符
□ 无"暂无数据"、"待补充"、"[待填充]"等空内容
□ 报告标题格式正确：# YYYYMMDD 每日复盘
```

**任何一项不通过，必须重新生成对应章节，不得推送。**
