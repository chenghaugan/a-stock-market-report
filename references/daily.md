**用途**：由 Hermes Agent 在完成数据采集后加载，用于驱动 Tavily 研究和完整 Markdown 报告生成。

**前置条件**：已完成 `run_report.py` 数据采集，JSON 数据文件位于 `{workdir}data_YYYYMMDD.json`。

```

## 报告信息

**⚠️ 以下变量在执行时必须替换为实际值**：
- `{date}` → 报告日期，格式 `YYYY-MM-DD`（如 `2026-04-22`）
- `{target_date}` → 数据日期，同上
- `{report_type}` → 报告类型：`日报`
- `{workdir}` → 工作目录：`{data_dir}`
- `{generated_at}` → 生成时间戳

```
今日日期（北京时间）：{date} 15:00 收盘后
报告类型：{report_type}
工作目录：{workdir}
JSON数据文件：{workdir}data_{date}.json（日期用数字格式：data_20260422.json）
Obsidian推送目标：
  - URL: https://obsidian-api.870314.top:5000
  - Vault: /01_投资研究/每日复盘
  - 子目录: 日报
  - 完整路径: /01_投资研究/每日复盘/日报/{date}_日报.md
  - 文件名: {date}_日报.md（例如：2026-04-22_日报.md）
  - 推送方式: 使用 push_to_obsidian.py（自动处理子目录）

**📊 数据显示格式说明（重要）**：
- `hot_sectors[].change` 和 `sectors[].change`：**已统一为百分比格式**
- 指数/个股涨跌幅：已经是百分比值（如 1.42 表示 1.42%），显示格式 `{value}%`
- 板块涨跌幅显示规则：
  - `change: 0.50` → 显示 `+0.50%`（正数）或 `-{abs(value)}%`（负数）
  - `change: 85.93` → 显示 `+85.93%`（已是百分比，无需转换）
- 模板中的 `{pct}` 占位符：填写 change 字段的绝对值（不加 ±）
- **绝对不要**对板块 change 值再做 ×100 运算

## 新闻数据存储（新增）

**⚠️ 重要：日报生成时，Tavily 搜索结果会自动保存到 `output/news/daily/` 目录**

### 存储机制

```
1. 日报生成时执行 Tavily 搜索（3-5 次）
2. 搜索结果自动保存到 output/news/daily/news_YYYYMMDD.json
3. 新闻数据格式：
   {
     "date": "2026-04-22",
     "search_queries": ["A股 今日行情...", ...],
     "results": [{"query": "...", "items": [...]}],
     "meta": {"total_items": 15, "credit_used": 3}
   }
```

### 好处

- **周报/月报复用**：避免重复搜索相同新闻，节省 Tavily credits
- **数据可追溯**：所有新闻数据持久化，便于回溯分析
- **历史对比**：可对比不同日期的新闻覆盖情况

## Tavily 研究策略

### 搜索原则

1. **搜索次数**：至少 **5-8 次** Tavily 搜索（含自选股新闻搜索）
2. **搜索深度**：`search_depth=basic`（常规），复杂行情用 `search_depth=advanced`
3. **时间范围**：
   - 大盘/板块新闻：`time_range=day`（当日）
   - **自选股新闻：`time_range=week`（过去7天，确保新闻能被索引）**
4. **主题**：`topic=finance`
5. **自选股新闻搜索**（新增）：针对涨幅/跌幅超3%的异动股单独搜索

### 数据采集失败兜底逻辑（重要）

**⚠️ 当 JSON 数据中 `quality_report.passed=false` 或 `quality_score < 0.5` 时，必须执行 Tavily 兜底搜索**

```
# 检查 JSON 中的 quality_report 字段
quality_report.passed == false  → 数据质量不通过，必须补充搜索
quality_report.quality_score < 0.5 → 触发兜底机制

# 兜底搜索 Query（从 tavily_supplement.queries 提取）
{
  "tavily_supplement": {
    "needed": true,
    "queries": ["A股行业板块涨幅排名 今日", "今日涨停板 涨停原因"],
    "fields": ["sectors", "zt_pool"]
  }
}

# 执行流程：
1. 读取 JSON → 检查 quality_report.tavily_supplement.needed
2. 如果 needed=true → 执行 tavily_supplement.quies 中的每个搜索
3. 搜索结果用于补充缺失章节（按 tavily_supplement.fields 对应）
```

**字段-章节映射**：
| tavily_supplement.fields | 对应章节 |
|--------------------------|----------|
| `indices` | 一、市场全景概览 |
| `sectors` | 三、五大热门板块深度解析 |
| `zt_pool` | 四、五大核心个股追踪 |
| `watchlist_a/watchlist_hk` | 五、自选股跟踪 |

## 报告写作规范

### 语言风格
- 分析师风格：专业、简洁、数据驱动
- 避免口语化表达
- 禁止使用占位符（"由AI深度分析后补充"、"待更新"、"暂无数据时请补充"）
- **如果数据确实为空**，如实标注"该数据暂无来源"

### 数据使用
- 优先使用 JSON 文件中的准确数字
- 涨跌幅保留2位小数（%）
- 成交额用"亿元"单位
- 点位取整数

### 禁止事项
- 禁止生成占位符
- 禁止编造未验证的数据
- 禁止提供具体投资建议
- **禁止省略表格分隔行**：所有表格必须包含 `| 表头 |` + `| --- |` + `| 数据 |`
